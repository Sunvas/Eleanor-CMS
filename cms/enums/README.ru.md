# Каталог PHP-перечислений

Данный каталог предназначен для размещения PHP-перечислений (`enum`), используемых в Eleanor CMS.

## Принцип работы автозагрузчика

Автозагрузчик обслуживает только пространство имён `CMS\Enums`. При запросе перечисления автозагрузчик преобразует имя перечисления в имя файла, приводя его к нижнему регистру и дополнительно проверяя kebab-case вариант имени. Например, запрос:

```php
CMS\Enums\TestEnum
```

приведёт к последовательной проверке файлов:

```text
testenum.php
test-enum.php
```

Поддерживается организация перечислений через подкаталоги. Каждый подкаталог соответствует дополнительному уровню пространства имён внутри `CMS\Enums`. Например, для перечисления:

```php
CMS\Enums\Test\SomeStatus
```

автозагрузчик выполнит поиск файлов:

```text
./test/somestatus.php
./test/some-status.php
```

---

## Правила и требования

### 1. Именование файлов

Автозагрузчик поддерживает оба варианта имён файлов:

```text
testenum.php
test-enum.php
```

Имена файлов регистрозависимы и должны быть записаны в нижнем регистре.

### 2. Независимость от namespace и имени перечисления

Автозагрузчик определяет соответствие перечисления исключительно по имени файла. Фактическое пространство имён и имя перечисления внутри файла не участвуют в поиске и могут быть произвольными. Благодаря этому каталог может использоваться как адаптерный слой для сторонних библиотек или внешних перечислений.

### 3. Возврат полного имени перечисления (FQCN)

Если перечисление внутри файла не соответствует ожидаемому имени перечисления в `CMS\Enums`, файл должен вернуть полное квалифицированное имя перечисления (FQCN):

```php
return \Vendor\Package\Status::class;
```

Пример:

```php
<?php

namespace Vendor\Package;

enum Status: string
{
    case ACTIVE='ACTIVE';
    case DISABLED='DISABLED';
}

return \Vendor\Package\Status::class;
```

После загрузки файла автозагрузчик:
- использует перечисление напрямую, если оно уже принадлежит пространству имён `CMS\Enums`;
- либо автоматически создаёт псевдоним внутри `CMS\Enums`.

### 4. Автоматическое создание псевдонима

Перечисление не обязано принадлежать пространству имён `CMS\Enums`. Если файл возвращает FQCN перечисления из другого пространства имён, автозагрузчик автоматически создаёт псевдоним внутри `CMS\Enums`, обеспечивая доступ к перечислению через единый механизм автозагрузки. Например:

```php
CMS\Enums\Test\SomeStatus
```

может фактически ссылаться на:

```php
\External\Library\StatusEnum
```

### 5. Необязательный return

Если перечисление уже объявлено в ожидаемом пространстве имён и его имя соответствует имени файла, файл может не возвращать FQCN. Например, файл:

```text
test/some-status.php
```

может напрямую содержать:

```php
<?php
namespace CMS\Enums\Test;

enum SomeStatus: string
{
    case ACTIVE='ACTIVE';
}
```

В этом случае автозагрузчик обработает перечисление автоматически без дополнительного `return`.
