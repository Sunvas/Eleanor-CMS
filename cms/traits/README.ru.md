# Каталог PHP-трейтов

Данный каталог предназначен для размещения PHP-трейтов (`trait`), используемых в Eleanor CMS.

## Принцип работы автозагрузчика

Автозагрузчик обслуживает только пространство имён `CMS\Traits`. При запросе трейта автозагрузчик преобразует имя трейта в имя файла, приводя его к нижнему регистру и дополнительно проверяя kebab-case вариант имени. Например, запрос:

```php
CMS\Traits\TestTrait
```

приведёт к последовательной проверке файлов:

```text
testtrait.php
test-trait.php
```

Поддерживается организация трейтов через подкаталоги. Каждый подкаталог соответствует дополнительному уровню пространства имён внутри `CMS\Traits`. Например, для трейта:

```php
CMS\Traits\Test\SomeFeatureTrait
```

автозагрузчик выполнит поиск файлов:

```text
./test/somefeaturetrait.php
./test/some-feature-trait.php
```

---

## Правила и требования

### 1. Именование файлов

Автозагрузчик поддерживает оба варианта имён файлов:

```text
testtrait.php
test-trait.php
```

Имена файлов регистрозависимы и должны быть записаны в нижнем регистре.

### 2. Независимость от namespace и имени трейта

Автозагрузчик определяет соответствие трейта исключительно по имени файла. Фактическое пространство имён и имя трейта внутри файла не участвуют в поиске и могут быть произвольными. Благодаря этому каталог может использоваться как адаптерный слой для сторонних библиотек или внешних трейтов.

### 3. Возврат полного имени трейта (FQCN)

Если трейт внутри файла не соответствует ожидаемому имени трейта в `CMS\Traits`, файл должен вернуть полное квалифицированное имя трейта (FQCN):

```php
return \Vendor\Package\SomeTrait::class;
```

Пример:

```php
<?php
namespace Vendor\Package;

trait SomeTrait
{
    protected function helper():void
    {
        // ...
    }
}

return \Vendor\Package\SomeTrait::class;
```

После загрузки файла автозагрузчик:

- использует трейт напрямую, если он уже принадлежит пространству имён `CMS\Traits`;
- либо автоматически создаёт псевдоним внутри `CMS\Traits`.

### 4. Автоматическое создание псевдонима

Трейт не обязан принадлежать пространству имён `CMS\Traits`. Если файл возвращает FQCN трейта из другого пространства имён, автозагрузчик автоматически создаёт псевдоним внутри `CMS\Traits`, обеспечивая доступ к трейту через единый механизм автозагрузки. Например:

```php
CMS\Traits\Test\SomeFeatureTrait
```

может фактически ссылаться на:

```php
\External\Library\FeatureTrait
```

### 5. Необязательный `return`

Если трейт уже объявлен в ожидаемом пространстве имён и его имя соответствует имени файла, файл может не возвращать FQCN. Например, файл:

```text
test/some-feature-trait.php
```

может напрямую содержать:

```php
<?php
namespace CMS\Traits\Test;

trait SomeFeatureTrait
{
    protected function helper():void
    {
        // ...
    }
}
```

В этом случае автозагрузчик обработает трейт автоматически без дополнительного `return`.
