[English](README.md)

В этом каталоге находятся юниты CMS — компоненты, реализующие функциональность сайта.

Каждый юнит представлен PHP-файлом вида `unit_name.php`. Этот файл должен возвращать объект. Стандартные точки входа CMS определяются интерфейсами, например:

- `CMS\Interfaces\UserArea` — юнит доступен из публичной пользовательской части.
- `CMS\Interfaces\AdminPanel` — юнит доступен из панели администратора.
- `CMS\Interfaces\Cron` — юнит может выполняться как фоновая задача.
- `CMS\Interfaces\CLI` — юнит может выполняться из командной строки.

Объект юнита может одновременно реализовывать несколько стандартных интерфейсов. Проекты также могут определять дополнительные интерфейсы и точки входа, если нужны собственные контексты выполнения.

Файл `unit_name.php` обычно должен оставаться небольшим и содержать только определение объекта юнита и диспетчеризацию точек входа. Более сложную логику можно размещать в необязательном каталоге `unit_name/`. В этом каталоге могут находиться обработчики панели администратора, обработчики пользовательской части, cron-логика, конфигурационные файлы, шаблоны, демонстрационные данные или любые другие файлы, необходимые юниту.

Например:

```text
blog.php
blog/
	admin-panel.php
	user-area.php
```

После загрузки юнита его объект сохраняется в глобальной переменной `$CMS` как динамическое свойство с тем же именем, что и юнит.

Шаблоны юнита рекомендуется размещать в каталогах шаблонов соответствующего контекста:

- `cms/admin-panel/unit_name/` — шаблоны для панели администратора.
- `cms/user-area/unit_name/` — шаблоны для публичной пользовательской части.

Статические файлы юнита должны размещаться в каталоге `static/unit_name/`.
